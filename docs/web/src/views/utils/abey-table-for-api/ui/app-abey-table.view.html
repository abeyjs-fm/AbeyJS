Ejemplo en vivo

Probá buscar artistas y cambiar de página: cada acción ejecuta la carga remota.

Cómo implementar el consumo de datos

La tabla no hace fetch sola: en modo flow pide datos al runtime con un intent; tu agente (o slice) responde publicando eventos con columnas, acciones e ítems. Paginación, tamaño de página y búsqueda vuelven a disparar ese intent con page, pageSize y query. Usá las pestañas para ver cada parte.

Herramientas que encajan (mapa mental)

  • <abey-table> + atributos flow, loadNetwork, intents y nombres de eventos.
  • bootstrapOmegaApp + globalThis.__abeyRuntime (lo expone exposeBootstrapRuntime).
  • registerAbeyJsUi() o AbeyTableElement.define("abey-table") antes de que el tag exista en el DOM.
  • Agente + behavior (reglas intent → acción) + channel.emit hacia los topics que la tabla escucha.
  • Repositorio HTTP (ej. DeezerArtistRepo) llamado desde el agente, no desde la plantilla.

Flujo en una frase

UI: usuario cambia página o busca → tabla dispatch Artist/TableLoad con payload → behavior enruta a loadTable → agente llama API → emit(eventTableColumns | eventTableActions | eventTableItems) → tabla actualiza filas y total.

Markup (esta demo)

flow="true" conecta el canal. loadNetwork="true" indica que filas y total vienen del servidor: no se re-filtra en cliente. Los intent* y event* deben coincidir con strings en tu semantics (aquí ArtistEcosystem). 

<abey-table
  selectable
  loadNetwork="true"
  flow="true"
  intentload="{{ intentLoad }}"
  intentselection="{{ intentSelection }}"
  intentaction="{{ intentAction }}"
  eventcolumns="{{ eventColumns }}"
  eventactions="{{ eventActions }}"
  eventitems="{{ eventItems }}"
  pageSizes="10,20,30"
  pageSize="10"
  page="1"
>
  <template data-abey-cell="name"> ... </template>
</abey-table>

Respuesta esperada por eventos

Publicá columnas y acciones una vez o en cada carga según tu modelo. Para ítems, incluí items, totalItems, page y pageSize para alinear paginador y datos. 

emit(eventItems, {
  items: [...],
  totalItems: 123,
  page: 1,
  pageSize: 10
});

Agente y registro Omega

En onAction("loadTable", payload) leé payload.page, payload.pageSize, payload.query. Los intents se registran con runtime.onIntent(...) y el ArtistBehavior traduce nombre de intent a acción ( Artist/TableLoad → loadTable). 

// omega/register.ts — installArtistOmega(runtime)
runtime.onIntent(ArtistEcosystem.intentLoadTable, (payload) => {
  void runtime.flow.handleIntent(
    intentOf(ArtistEcosystem.intentLoadTable, payload),
    { source: "artist-ecosystem" },
  );
});

Búsqueda

El buscador de la toolbar dispara la recarga cuando el texto se estabiliza (debounce, configurable con AbeyTableElement.searchDebounceMs), al salir del campo, al clic fuera o con Enter si quedaba un disparo pendiente. Opcionalmente podés usar un intent aparte (intentsearch="...") si querés separar búsqueda de la carga de tabla.
