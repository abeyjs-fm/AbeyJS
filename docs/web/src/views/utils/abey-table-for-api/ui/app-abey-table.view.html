Live example

Try searching for artists and changing pages: each action triggers a remote load.

How remote data feeding works

The table does not fetch on its own: in flow mode it asks the runtime for data with an intent; your agent (or slice) answers by emitting events with columns, actions, and items. Pagination, page size, and search fire that intent again with page, pageSize, and query. Use the tabs to explore each piece.

Pieces that fit together

  • <abey-table> + flow, loadNetwork, intent names and event names.
  • bootstrapOmegaApp + globalThis.__abeyRuntime (via exposeBootstrapRuntime).
  • registerAbeyJsUi() or AbeyTableElement.define("abey-table") before the custom element enters the DOM.
  • Agent + behavior (intent → action rules) + channel.emit to topics the table listens to.
  • HTTP repo (e.g. DeezerArtistRepo) called from the agent, not from the template.

Flow in one sentence

UI: user changes page or searches → table dispatches Artist/TableLoad with payload → behavior routes to loadTable → agent hits the API → emit(eventTableColumns | eventTableActions | eventTableItems) → table updates rows and total.

Markup (this demo)

flow="true" attaches the channel. loadNetwork="true" means rows and total come from the server: rows are not re-filtered locally. intent* and event* must match strings from your semantics (here ArtistEcosystem). 

<abey-table

  selectable

  loadNetwork="true"

  flow="true"

  intentload="{{ intentLoad }}"

  intentselection="{{ intentSelection }}"

  intentaction="{{ intentAction }}"

  eventcolumns="{{ eventColumns }}"

  eventactions="{{ eventActions }}"

  eventitems="{{ eventItems }}"

  pageSizes="10,20,30"

  pageSize="10"

  page="1"

>

  <template data-abey-cell="name"> ... </template>

</abey-table>

What to emit

Publish columns and actions once per session or every load depending on your model. For items, include items, totalItems, page, and pageSize so the pager stays in sync with the data. 

emit(eventItems, {

  items: [...],

  totalItems: 123,

  page: 1,

  pageSize: 10

});

Agent and Omega registration

In onAction("loadTable", payload), read payload.page, payload.pageSize, payload.query. Intents are wired with runtime.onIntent(...) and ArtistBehavior translates the intent name to an action ( Artist/TableLoad → loadTable). 

// omega/register.ts — installArtistOmega(runtime)

runtime.onIntent(ArtistEcosystem.intentLoadTable, (payload) => {

  void runtime.flow.handleIntent(

    intentOf(ArtistEcosystem.intentLoadTable, payload),

    { source: "artist-ecosystem" },

  );

});

Search

The toolbar search reloads once the text stabilizes (debounced; tune via AbeyTableElement.searchDebounceMs), on blur, outside click, or Enter when a pending dispatch was queued. Optionally use a separate intent (intentsearch="...") to decouple search from table load.
