Implementation Guide

The student CRUD uses a decoupled architecture where the UI communicates with the state through intents. Explore the tabs to understand the technical structure.

CRUD Flow

  • Listing: The abey-table component visualizes the students collection.
  • Create/Edit: The open-create or edit intent is triggered, which activates the modal.
  • Dynamic Form: The abey-form is automatically generated and validates data before emitting the save action.

Note: As a "memory-only" example, changes persist in the global runtime state but are lost on page refresh.

<!-- Table with local actions -->

Omega Interaction

The component communicates with the Agent via business intents.

// Agent action handler example
onAction("saveStudent", (student) => {
  if (student.id) {
    this.repo.update(student);
  } else {
    this.repo.create(student);
  }
  this.channel.emit("students/changed");
});
